# Neovim Config - Claude Code Preferences

> **IMPORTANT**: Do NOT edit this file unless the user explicitly asks to update CLAUDE.md.

## Architecture
- LazyVim-based config on the `main` branch (uses `nvim-treesitter` main branch API)
- Plugin configs live in `lua/plugins/`
- Custom snippets (VSCode JSON format) live in `snippets/`

## Performance Rules
- All changes MUST be performance-safe. No heavy per-keystroke computations.
- Prefer in-process solutions (snippets, treesitter) over external LSP servers when possible
- Disabled plugins for performance: noice, nvim-lint, gitsigns, bufferline, persistence, mini.ai, treesitter-context
- Lualine throttled to 500ms refresh

## Completion (blink.cmp)
- Sources: lsp (score 100), snippets (score 90), path, buffer
- Tab/S-Tab unbound (use Enter to accept, Ctrl-n/p to navigate)
- Ghost text disabled
- Completions disabled in comments and Avante buffers
- Emmet LSP only for HTML files (NOT for TSX/JSX - causes unwanted suggestions)
- Bracket-only completions from emmet are filtered out
- Do NOT override snippets provider `opts.search_paths` - blink.cmp defaults already include `~/.config/nvim/snippets/`

## Snippets
- Use blink.cmp built-in snippets provider (NOT LuaSnip)
- VSCode JSON format in `snippets/` directory
- `package.json` maps snippet files to filetypes
- Do NOT use nested tabstops like `${1: className='$2'}` - causes LSP snippet grammar errors
- Keep snippet files minimal - only add frequently used patterns
- Snippets config consolidated in `performance.lua` (not separate snippets.lua)

## LSP
- emmet_ls: HTML only (not TSX/JSX)
- eslint: disabled (TypeScript sufficient)
- vtsls: TypeScript LSP with 200ms debounce
- Custom `gd` mapping filters out node_modules

## Treesitter
- textobjects configured via `nvim-treesitter-textobjects` plugin (NOT through nvim-treesitter opts)
- LazyVim only auto-creates keymaps for `move` - `select` keymaps (af/if/ac/ic/aa/ia) MUST be created manually via lazy.nvim `keys` spec
- Uses LazyVim's `keys` property under `move` for buffer-local keymaps
- `[f`/`]f` navigation handled by textobjects, NOT custom keymaps in keymaps.lua
